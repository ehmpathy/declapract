# blocker.1: Test uses try/catch without rethrowing unexpected errors

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.test/pitofsuccess.errors/rule.forbid.failhide.md

**locations**:
- src/domain.operations/usage/evaluate/projectVariableExpressions/replaceProjectVariablesInDeclaredFileContents.test.ts:24

The test employs a try/catch block to assert that an error is thrown, but it catches all errors without rethrowing unexpected ones. This violates the failhide rule, as it could silently pass if an unexpected error is thrown (though the assertion would fail). Instead, use Jest's expect().toThrow() for cleaner, failfast testing that doesn't catch unexpectedly.
