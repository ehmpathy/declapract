# blueprint: array variable support

## summary

add array variable support to `replaceProjectVariablesInDeclaredFileContents`:
1. preserve arrays through `flatten()` via `{ safe: true }` option
2. detect arrays via `Array.isArray()` and serialize via `JSON.stringify()`
3. add test coverage for array variable replacement

the change is localized to one function and its test file.

### test cases to add

**primary test case:**

```typescript
const exampleResourcesFileContents = `
reviewers: { users: @declapract{variable.productionApprovers}, teams: null },
origins: @declapract{variable.allowedOrigins},
`.trim();

it('should replace array variables with JSON array literals', () => {
  const replacedFileContents = replaceProjectVariablesInDeclaredFileContents({
    projectVariables: {
      productionApprovers: ['alice', 'bob'],
      allowedOrigins: ['https://example.com', 'https://api.example.com'],
    },
    fileContents: exampleResourcesFileContents,
  });
  expect(replacedFileContents).toEqual(
    `
reviewers: { users: ["alice","bob"], teams: null },
origins: ["https://example.com","https://api.example.com"],
    `.trim(),
  );
});
```

**edge case tests (optional):**

## implementation details

### type change

```typescript
// before
const flattenedProjectVariables: Record<string, string> =
  flatten(projectVariables);

// after
const flattenedProjectVariables: Record<string, string | string[]> =
  flatten(projectVariables, { safe: true });
```

### replacement logic

```typescript
// before
return replaceAll(contents, thisVariableExpression, variableValue);

// after
const replacement = Array.isArray(variableValue)
  ? JSON.stringify(variableValue)
  : variableValue;
return replaceAll(contents, thisVariableExpression, replacement);
```

## backwards compatibility

| scenario | before | after | compatible? |
|----------|--------|-------|-------------|
| string variable | `"value"` | `"value"` | yes |
| nested object | `"nested.key"` access | `"nested.key"` access | yes |
| array variable | not supported | `["a","b"]` | yes (additive) |

the change is fully backwards compatible — all extant string and nested object variables work as before.
