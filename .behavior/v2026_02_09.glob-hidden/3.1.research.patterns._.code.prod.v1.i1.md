# research: prod codepath patterns for glob hidden directory fix

## overview

declapract uses glob patterns to match files for practice checks. the glob call occurs in one primary location, with one utility function that also uses glob.

## pattern 1: evaluateProjectAgainstFileCheckDeclaration [REPLACE]

**file**: `src/domain.operations/usage/evaluate/evaluateProjectAgainstFileCheckDeclaration.ts` [1]

**current implementation**:
```ts
import glob from 'fast-glob';

const pathsFoundByGlob = await withDurationReporting(
  `glob:${check.pathGlob}`,
  () =>
    glob(check.pathGlob, {
      cwd: project.getProjectRootDirectory(),
      dot: true,
      onlyFiles: true,
      ignore: ['node_modules'],
    }),
)();
```

**what it does**: resolves `check.pathGlob` to actual file paths in the project directory [1]

**the issue**: `fast-glob` with `dot: true` does not traverse hidden directories in globstar expansion [2]

**action**: [REPLACE] — switch from `fast-glob` to `node:fs/promises` glob

**new implementation**:
```ts
import { glob } from 'node:fs/promises';

const pathsFoundByGlob = await withDurationReporting(
  `glob:${check.pathGlob}`,
  async () => {
    const matches: string[] = [];
    for await (const entry of glob(check.pathGlob, {
      cwd: project.getProjectRootDirectory(),
      exclude: (name) => name === 'node_modules',
    })) {
      matches.push(entry);
    }
    return matches;
  },
)();
```

**notes**:
- node.js `fs.glob` fixed globstar + hidden dir traversal in PR #61012 [3]
- `fs.glob` returns an async iterator, not a promise of array
- `exclude` option replaces `ignore` (function-based)
- no `dot` option needed — built-in handles hidden dirs correctly
- no `onlyFiles` option — need to filter if required

## pattern 2: listFilesInDirectory [REPLACE]

**file**: `src/utils/filepaths/listFilesInDirectory.ts` [4]

**current implementation**:
```ts
import glob from 'fast-glob';

export const listFilesInDirectory = async ({
  directory,
}: {
  directory: string;
}) => glob('**/*', { cwd: directory, dot: true, onlyFiles: true });
```

**what it does**: lists all files recursively in a directory [4]

**action**: [REPLACE] — switch from `fast-glob` to `node:fs/promises` glob

**new implementation**:
```ts
import { glob } from 'node:fs/promises';
import { stat } from 'node:fs/promises';
import { join } from 'node:path';

export const listFilesInDirectory = async ({
  directory,
}: {
  directory: string;
}): Promise<string[]> => {
  const matches: string[] = [];
  for await (const entry of glob('**/*', { cwd: directory })) {
    const fullPath = join(directory, entry);
    const stats = await stat(fullPath);
    if (stats.isFile()) matches.push(entry);
  }
  return matches;
};
```

**notes**:
- need to manually filter for files since `fs.glob` lacks `onlyFiles` option
- alternatively, could use `**/*.{*,}` pattern if extension filter is acceptable
