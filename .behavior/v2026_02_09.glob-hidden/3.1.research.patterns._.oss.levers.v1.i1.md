# research: oss levers for glob hidden directory traversal

## context

declapract currently uses `fast-glob@3.2.2` with `dot: true` option [1]. the issue is that `**` globstar does not traverse hidden directories (directories that start with `.`) even when `dot: true` is set [2].

## oss tools available

### 1. fast-glob (current)

**repo**: [mrmlnc/fast-glob](https://github.com/mrmlnc/fast-glob) [3]

**maintenance**: actively maintained, 1.5k+ stars, used by major tools

**the limitation**: the `dot` option controls whether dotfiles are matched in results, but does not control whether hidden directories are traversed in globstar expansion [4]:

> "An asterisk (*) — matches all except slashes (path separators), hidden files (names that start with `.`)." [4]

> "A double star or globstar (`**`) — matches zero or more directories." [4]

the `dot: true` option allows matched dotfiles in results, but `**` still skips hidden directories in traversal.

**workaround pattern**: expand glob patterns to explicitly include hidden directory variants [5]:

```ts
// original pattern
'.accept.blackbox/**/*.ts'

// expanded patterns
[
  '.accept.blackbox/**/*.ts',           // original (misses nested hidden dirs)
  '.accept.blackbox/**/.*/**/*.ts',     // explicit hidden dir traversal
]
```

**pros**:
- already in use, no new dependency
- fast performance (benchmark leader) [3]
- well maintained

**cons**:
- requires pattern expansion workaround
- workaround adds complexity

---

### 2. globby

**repo**: [sindresorhus/globby](https://github.com/sindresorhus/globby) [6]

**maintenance**: actively maintained, 2.5k+ stars, widely used

**behavior**: wrapper around fast-glob [6]:

> "Globby exhibits all of the same pattern semantics as fast-glob, as it is a wrapper around fast-glob." [7]

**features**:
- automatic `.gitignore` respect
- negated globs via `!` in patterns
- slightly higher-level api

**pros**:
- more ergonomic api
- gitignore integration

**cons**:
- same core limitation as fast-glob (wrapper, not fix)
- 10-20% slower than fast-glob [7]
- adds no value for this specific issue

---

### 3. tinyglobby

**repo**: [SuperchupuDev/tinyglobby](https://github.com/SuperchupuDev/tinyglobby) [8]

**maintenance**: newer library, adoption is on the rise

**behavior**: alternative to globby and fast-glob, has `dot` option to "allow entries that start with a period" [8]

**pros**:
- smaller bundle size
- modern implementation

**cons**:
- same globstar limitation applies (standard glob semantics)
- less battle-tested than fast-glob
- no evidence of a fix for this specific issue

---

### 4. node-glob (isaacs)

**repo**: [isaacs/node-glob](https://github.com/isaacs/node-glob) [9]

**maintenance**: actively maintained, 8k+ stars, the original node glob

**behavior**: provides `dot` option [9]:

> "The `dot` option can be used to include .dot files in normal matches and globstar matches." [9]

**note**: documentation suggests `dot` affects globstar matches, but real-world behavior matches fast-glob limitation [2]

**pros**:
- widely adopted
- comprehensive options

**cons**:
- same core globstar limitation
- slower than fast-glob [7]

---

### 5. node.js built-in fs.glob

**behavior**: node.js has built-in `fs.glob()` since v22 [2]

**the issue**: same globstar limitation was reported [2]:

> "Whenever `**` is used in the pattern, the dot (hidden) files are not matched, even though the pattern specifically requests them." [2]

**resolution**: issue was closed as **COMPLETED** with pull request #61012 that implements a fix to detect dot files with globstar patterns [2].

**pros**:
- no external dependency
- fix is merged

**cons**:
- requires node v22+
- declapract supports node >=18.19.0 [1]
- compat break to use built-in over fast-glob

---

## recommendation

### option a: pattern expansion with fast-glob (recommended)

stay with fast-glob, implement pattern expansion workaround:

```ts
const expandGlobForHiddenDirs = (pattern: string): string[] => {
  if (!pattern.includes('**')) return [pattern];
  return [
    pattern,
    pattern.replace('**/', '**/.*/**/'),
  ];
};
```

**rationale**:
- no new dependencies
- minimal code change
- preserves fast performance
- battle-tested library

### option b: switch to node.js built-in fs.glob

requires bump of minimum node version to 22+.

**rationale**:
- native solution
- no external dependency
- officially fixed

**tradeoff**: breaks compat with node 18/20 users

### option c: custom directory walker

implement manual recursive directory traversal that explicitly enters hidden directories.

**rationale**:
- full control
- no library limitations

**tradeoff**: more code, more maintenance, wheel reinvention
