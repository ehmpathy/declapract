# vision: glob matches files in nested hidden directories

## approach

switch from `fast-glob` to node.js built-in `fs.glob` (available in node 22+). the built-in implementation has a fix for globstar + hidden directory traversal (PR #61012). we can require node 22+ — no compat concerns.

**why fs.glob**:
- native solution, no external dependency
- officially fixed for this exact issue
- simplifies dependency tree
- future-proof (node maintains it)

## the outcome world

### before

a developer defines a declapract practice that checks test assets:

```yaml
pathGlob: '.accept.blackbox/**/*.ts'
```

their project has this structure:
```
.accept.blackbox/
  foo.ts                    # matched ✓
  bar/
    baz.ts                  # matched ✓
  .test/
    assets/
      example.ts            # silently skipped ✗
```

the developer assumes all `.ts` files under `.accept.blackbox/` are checked. they're wrong. files inside `.test/` are silently ignored. the practice passes — but it's a false positive. hidden directory contents are invisible to the glob.

### after

the same pattern now works as expected:

```
.accept.blackbox/
  foo.ts                    # matched ✓
  bar/
    baz.ts                  # matched ✓
  .test/
    assets/
      example.ts            # matched ✓
```

no special syntax required. `**` traverses hidden directories just like visible ones. the glob behaves as developers intuit it should.

### the "aha" moment

> "wait, so before this fix, my `.hidden/**/*` pattern was only matching top-level dotfiles, not the whole tree? that explains why my test fixtures weren't being validated!"

the value clicks when developers realize their patterns were silently incomplete — and now they're not.

## user experience

### usecases

1. **test asset validation** — practices that check `.test/` or `.fixtures/` directories nested inside other hidden dirs
2. **hidden config hierarchies** — projects with `.config/.local/settings.json` structures
3. **agent/role definitions** — patterns like `.agent/**/*.md` that should traverse `.agent/role=x/.hidden/`

### contract inputs & outputs

**input**: unchanged — developers write glob patterns as before
```yaml
pathGlob: '.accept.blackbox/**/*.ts'
```

**output**: files in nested hidden directories are now included in matches

### timeline

1. developer writes practice with `**` pattern
2. declapract evaluates project
3. all matching files — including those in nested hidden dirs — are checked
4. no surprises, no silent omissions

## mental model

### how users describe it to a friend

> "declapract globs now work how you'd expect — `**` means *all* subdirectories, including hidden ones. before there was this weird edge case where nested dotfolders were skipped."

### analogies

- like `find . -name "*.ts"` — it doesn't skip `.hidden/` directories
- like `ls -laR` — recursive listing includes everything
- unlike the old behavior — which was more like `ls -R` (skipping dotfiles unless you ask)

| "hidden folders" | "dot-prefixed directories" |
| "nested dotdirs" | "nested hidden directories" |

## evaluation

### how well does it solve the goals?

| goal | status |
|------|--------|
| match files in nested hidden dirs | ✓ solved |
| no breaking changes to extant patterns | ✓ patterns behave as expected, just more complete |
| backwards compatible | ✓ prior matches still work, new matches are added |

### pros

- **intuitive behavior** — `**` means all, not "all except hidden subdirs"
- **no config needed** — developers don't need to learn workarounds
- **pit of success** — the obvious pattern is the correct pattern

### cons

- **slightly more matches** — patterns that accidentally relied on hidden dirs to be skipped will now match more files (this is a feature, not a bug)
- **node 22+ required** — bumps minimum node version (acceptable tradeoff for native fix)

### edgecases

| edgecase | behavior |
|----------|----------|
| `.hidden/.hidden/.hidden/file.ts` | matched ✓ (multiple nesting levels work) |
| `.hidden/visible/.hidden/file.ts` | matched ✓ (alternating hidden/visible works) |
| `node_modules/.bin/something` | not matched (node_modules still ignored) |
| pattern without `**` | unchanged behavior |

### pit of success

- developers write the pattern they intuit
- no special syntax or flags to remember
- hidden directories are first-class citizens in traversal
